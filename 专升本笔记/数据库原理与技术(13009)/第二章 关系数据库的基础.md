关系模型
基本术语:
	关系模型：用二维表格表示实体集，用关键码表示实体之间联系的数据模型
	关系模式：对关系模型的语义描述，关系模式就是关系名及所含属性的集合。
	==一列为一个属性,一行为一个记录也成为元组==
	基本术语:
		关键码:
			能够唯一标识一个元组中的一个属性和多个属性
		超键:
			超键是唯一找到某条数据的所有可能组合
				比如:{学号,姓名,年龄}可以唯一标识一个人,但是是学号也可以唯一标识一个人,超键可以有候选键组成
		候选键:
			没有多余属性的超键称为候选键
				比如:{sno}是候选键,{sno,sname}就不是候选键
		主键:从多个候选键选出一个最好用的,并且主键具有==唯一性==
			例如:身份证和学号都可以找到一个学生,但是选学号为主键,因为学号相对来说,比较简短,方便查询
		外键:
			如果关系模式R中的属性K是另一个模式T的主键，则称属性K是模式R的外键，此时称T为主表，R为附表。
完整性规则:
	实体完整性
		主键不能为空
	参照完整性
		确保关系数据库中的数据表间的关系始终保持一致性
	用户定义完整性
		根据关系数据库应用环境的不同,可以自定义约束条件
数据模型的三个重要组成部分:
	数据结构
		数据库中全部数据及其相互联系都被表示为“关系” (二维表格) 的形式
	数据操纵(DML)
		关系模型提供一组完备的高级关系运算，以支持对数据库的各种操作。
		数据查询:
			关系代数语言:查询操作以集合操作为基础的运算。
				==并==:关系S1和关系S2的并由属于S1或属于S2的元组组成，S1和S2的所有元组合并，删去重复元组，组成一个新关系
				==差==:在S1不在S2中的元组
				==交==:关系S1与关系S2的交由既属于S1又属于S2的元组组成，即S1与S2中相同的元组,组成一个新关系
				==笛卡尔积==
```
				假设有两个域：A= {熊大，熊二}，B= {吃饭，睡觉，打豆豆}，则A、B的笛 卡尔积的基数为2×3=6，即该笛卡尔积有6个元组。
				A×B= { (熊大，吃饭) ，
				(熊大，睡觉) ，
				(熊大，打豆豆)
				(熊二，吃饭) ，
				(熊二，睡觉) ，
				(熊二，打豆豆) }
```
			关系演算语言:查询操作以谓词演算为基础的运算。
			关系逻辑语言:查询操作以if一then逻辑操作为基础的运算
		数据更新
	数据完整性规则
		数据库中数据必须满足实体完整性、参照完整性和用户定义的完整性规则
与其他数据模型相比,关系模型的优点如下:
	关系模型提供单一的数据结构形式，具有高度的简明性和精确性。
	关系模型的逻辑结构和相应的操作完全独立于数据存储方式，数据独立性高。
	关系模型使数据库的研究建立在比较坚实的数学基础上。
关系运算:
	选择 ( σ)：找出满足条件的元组
	 投影 (∏) ：找出满足条件的属性
	 连接 (⋈  ) ：
		l θ连接：从两个关系的笛卡尔积中，选择属性间满足一定条件的元组
		l 等值连接： θ取“=”时为等值连接	
		l 自然连接：自然连接是一种特殊的等值连接，它要求两个关系中进行比较 的分量必须是==相同的属性组==，并且在结果中把==属性重复的列去掉==
例题:
为什么关系中的元组没有先后顺序且不允许有重复元组？
答案：因为关系是一个集合，元组是集合中的元素。而集合中的元素是没有先后顺序且不允许有重 复的。

外键值何时允许为空？何时不允许为空？
答案：当外键要引用参照关系的主键时，不允许为空。否则，可以为空
